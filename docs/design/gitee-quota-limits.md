# Gitee (码云) 额度与限制调研报告

## 1. 仓库与文件容量限制

### 1.1 仓库容量 (Repository Size)
*   **单仓库上限**:
    *   **免费版 (社区版)**: **500 MB** (强烈建议控制在 this 以内，超过可能无法 push 或被锁定)。
    *   **SVN 仓库**: **400 MB**。
    *   **付费/企业版**: 可升级至 1GB, 2GB, 3GB 等。
*   **仓库数量**: 免费版可创建 1000 个仓库 (不限公有/私有)。

### 1.2 文件大小限制 (File Size)
*   **Web 端上传**: 单文件最大 **50 MB** (免费版) / 100 MB (付费版)。
*   **Git 推送 (Push)**:
    *   **免费版**: 单文件最大 **100 MB**。
    *   **付费版**: 单文件最大 200 MB。
    *   **GVP (最有价值开源项目)**: 单文件最大 300 MB。
    *   *注意*: 超过 50MB 会有警告，超过 100MB 会直接拒绝 push (除非使用 LFS，但免费版 LFS 受限)。

### 1.3 Release (发行版) 与附件
*   **Release 附件大小**:
    *   **免费版**: 单个附件最大 **100 MB**。
    *   **GVP 项目**: 单个附件最大 200 MB。
*   **仓库附件总容量**:
    *   **免费版**: 全仓库附件总和不超过 **1 GB**。
    *   **推荐项目**: 5 GB。
    *   **GVP 项目**: 20 GB。

### 1.4 Git LFS (大文件存储)
*   **限制**: Gitee 的 LFS 功能主要面向**付费企业版**开放。
*   **现状**: 免费个人版用户很难使用 LFS，或者配额极低（部分旧资料显示需申请或不可用，官方文档提示“目前该功能针对付费企业开放”）。
*   **对比**: 这一点与 GitHub (免费用户也有 1GB LFS 存储 + 1GB 带宽) 差异巨大。

---

## 2. API 调用频率限制 (Rate Limits)

Gitee 的 API 限制比 GitHub 严格且计算方式不同（基于 IP 而非 Token 的部分更多）。

*   **API 请求**:
    *   **单 IP 限制**: **180 次 / 3分钟** (约 1次/秒)。
    *   *注意*: 这是一个相对较低的频率，高并发同步时极易触发。
*   **Web 页面访问**:
    *   **单 IP 限制**: **60 次 / 3分钟**。
*   **Token 限制**: Gitee OAuth 文档未明确公开基于 Token 的独立高额度，通常受限于 IP 频率。

---

## 3. Pages 服务限制 (静态网站托管)

*   **现状**: **非常不稳定 / 准暂停状态**。
*   **实名认证**: 必须进行“身份证 + 手持照片”实名认证才能开通。
*   **新规**: 许多新用户反馈无法开通 Pages 服务，提示“维护中”或“升级中”。
*   **Pro 版**: Gitee Pages Pro (自定义域名、自动更新) 已对个人用户下线，仅企业版可用。
*   **内容审查**: 极其严格，包含“敏感词”会被立刻封禁 Pages 甚至仓库。

---

## 4. 下载与访问限制 (防滥用)

*   **Raw 文件下载**:
    *   **大于 1MB 的文件**: 在未登录状态下访问 `raw` 链接（如图片、代码文件）可能会被重定向到登录页，导致图床失效。
    *   **外链限制**: 严厉打击将 Gitee 作为图床或网盘使用，可能会开启防盗链。
*   **下载速度**:
    *   国内访问速度极快，无带宽人为限制，但受限于服务器负载。

---

## 5. 总结：Gitee 适合做 Release 分发吗？

| 特性 | Gitee 情况 | 结论 |
| :--- | :--- | :--- |
| **Release 附件大小** | 100 MB | **适合** (绝大多数软件够用) |
| **仓库总容量** | 500 MB (代码) / 1GB (附件) | **需注意** (长期大量发版可能超标) |
| **API 稳定性** | 180次/3分钟 | **一般** (需控制同步脚本频率) |
| **公开下载体验** | 极快，但需登录 | **优秀** (国内用户体验好，但作为图床/Raw引用不佳) |
| **自动化同步** | 无官方 Action | **较难** (需自建脚本) |

**建议**:
Gitee 非常适合作为 **GitHub Release 的国内镜像下载源**（手动或脚本同步），但不适合作为唯一的“源头”仓库（因为 LFS 和 Pages 限制）。
