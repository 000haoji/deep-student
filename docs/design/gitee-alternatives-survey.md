# Gitee 替代方案调研报告 (Release 分发)

除了 Gitee，国内还有多种方案可用于 GitHub Release 的加速分发。以下是针对不同场景的替代方案深度调研。

## 1. 国内代码托管平台 (Code Hosting)

这些平台体验最接近 GitHub/Gitee，适合作为镜像仓库。

### 1.1 Coding.net (腾讯云开发者平台)
*   **现状**: 功能强大（含 DevOps/制品库），但运营策略正在调整。
*   **额度**:
    *   **免费版**: 团队成员 < 5 人免费。
    *   **制品库**: 免费版提供一定额度的制品存储（通常 1-10GB 级别），下载速度快（腾讯云 CDN）。
*   **风险**: **2025年政策变动**。公告显示 2025 年 9 月将下线"标准版"并调整免费额度，未来可能转向全面付费或大幅缩减免费权益。
*   **适用性**: 短期可用，长期存在不确定性。

### 1.2 GitLink (确实开源)
*   **背景**: CCF（中国计算机学会）官方平台，依托国防科大 Trustie 项目。
*   **额度**:
    *   **附件限制**: 单文件通常限制在 **50MB - 100MB** (参考 Gitea/Git 协议标准限制)。
    *   **带宽**: 无明确限制，国内教育网/公网访问速度较好。
*   **自动化**: API 生态较弱，缺乏现成的 GitHub Action，需自行开发同步脚本。
*   **适用性**: 适合学术类、教育类开源项目，或追求稳定的项目。

### 1.3 GitCode (CSDN)
*   **背景**: CSDN 旗下的代码托管平台。
*   **额度**:
    *   **单文件**: **100 MB** (Git 推送限制)。
    *   **仓库**: 免费版通常 1GB - 2GB。
*   **体验**: 商业化推广较多，界面广告较多。
*   **适用性**: 如果你的用户群体主要混迹于 CSDN，可以考虑。

### 1.4 AtomGit (开放原子)
*   **背景**: 开放原子开源基金会（OpenAtom）牵头。
*   **现状**: 平台较新，基础设施正在完善中。
*   **API**: OpenAPI 尚在完善，自动化工具链（如 CI/CD 集成）不如 Gitee/Coding 成熟。
*   **适用性**: 适合参与开放原子基金会孵化项目的开发者。

## 2. 网盘/云存储方案 (Netdisk & Object Storage)

适合纯二进制文件（Installer/Zip）的分发，速度通常快于代码托管平台。

### 2.1 123 云盘 (123pan.com)
*   **优势**: **不限速**（上传/下载均快），免登录下载（需分享设置）。
*   **限制 (2024 新规)**:
    *   **免费用户**: 单日非登录下载流量限制（约 **1GB/天**），超出需付费或登录。
    *   **直链**: 提供直链服务（WebDAV/API），但**按流量计费**（~0.05元/GB）。
*   **自动化**: 有第三方 Python SDK / API 封装，但官方 API 门槛较高（需开发者认证）。
*   **结论**: 适合**低频、大文件**分发。高频下载容易触发流量限制。

### 2.2 蓝奏云 (lanzou.com)
*   **优势**: **无限流量**，下载速度极快，适合小文件。
*   **致命限制**:
    *   **单文件限制**: 普通用户 **< 100MB** (非常严格)。
    *   **格式限制**: 不支持 apk/exe 等特定格式（需改后缀或压缩）。
*   **自动化**: 有非官方 API 库，但依赖网页逆向，稳定性一般。
*   **结论**: 仅适合**小型工具**或**安卓应用**（需改名）分发，不适合大型桌面软件。

### 2.3 Cloudflare R2 (推荐)
*   **背景**: Cloudflare 推出的对象存储（兼容 S3 API）。
*   **额度**:
    *   **存储**: **10 GB** 免费。
    *   **流量**: **零出口费用** (Free Egress) —— 这是最大的杀手锏。
*   **国内体验**:
    *   `*.r2.dev` 域名在国内部分被阻断。
    *   **解决方案**: 绑定**自定义域名** (Custom Domain)，国内访问速度尚可（虽然不如阿里云 OSS，但比 GitHub 快且稳）。
*   **自动化**: 完美兼容 AWS S3 SDK，GitHub Actions 插件丰富 (`rclone` 或 `s3-upload`)。
*   **结论**: **最佳开发者方案**。免费、可自动化、全球可用（含国内）。

## 3. 综合推荐与决策树

### 你的 Release 文件多大？
*   **< 100 MB**:
    *   **Gitee**: 首选 (速度快，免费)。
    *   **蓝奏云**: 备选 (速度极快，需处理文件名)。
*   **> 100 MB**:
    *   **123 云盘**: 适合手动上传分享，用户量不大时体验好。
    *   **Cloudflare R2**: 适合自动化分发，需绑定域名。
    *   **阿里云 OSS / 腾讯云 COS**: 适合**有预算**的企业/项目 (按流量付费，速度最快)。

### 是否需要自动化 (GitHub Action)?
*   **必须自动化**:
    *   **Gitee**: 使用 `action-gitee-release` (注意 100MB 限制)。
    *   **Cloudflare R2**: 使用 S3 兼容 Action 上传，极度灵活。
    *   **OSS/COS**: 官方 Action 支持完善。
*   **手动维护 OK**:
    *   **123 云盘 / 夸克 / 百度网盘**: 适合作为补充下载渠道。

### 终极建议组合
1.  **自动化主渠道**: **Gitee** (同步 Release) + **Cloudflare R2** (自定义域名直链)。
2.  **大文件兜底**: **123 云盘** (手动上传大版本安装包)。
