## 1. 间隔重复系统的用户体验设计

### 1.1 算法现状

| 算法 | 核心特征 | 性能 |
|------|----------|------|
| SM-2（Anki 默认） | 固定公式，手动参数 | 基线 |
| FSRS（Anki 可选） | 机器学习自适应，个性化记忆衰减曲线 | 91.9% 场景优于 SM-2，减少 20-30% 复习量 |
| SM-18（SuperMemo） | 偏差容忍，鲁棒性更强 | 相同保留率下复习量更少 |

**来源**: Anki Manual (docs.ankiweb.net), MintDeck Blog (FSRS benchmarks), SuperMemo CSEDU 2025 论文

**关键洞察**: Nate Meyvis 的研究指出，**UX 改进比算法优化带来更大的时间节省**——在制卡界面上省下 5 秒比在调度算法上省下 5 秒更容易实现。

### 1.2 到期提醒策略

| 策略 | 优势 | 劣势 | 适用场景 |
|------|------|------|----------|
| **系统托盘常驻图标** | 不打断工作流，持续可见 | 容易被忽略 | 桌面应用首选 |
| **应用内横幅** | 上下文相关，不侵入 | 仅在应用内可见 | 配合托盘使用 |
| **推送通知** | 高触达率 | 引起焦虑，被静音 | 移动端；桌面端慎用 |
| **系统托盘 + 气泡** | 温和提醒，支持一键进入 | 需要 OS 权限 | **推荐方案** |

**Duolingo 策略**: 将通知作为习惯循环的"触发器"（cue），但经过 600+ 次 A/B 测试优化通知频率和措辞。

**建议**: 桌面应用采用**系统托盘图标 + 到期数字徽标**，仅在应用启动时显示应用内横幅，避免弹窗式推送。

### 1.3 复习债务恐惧的解决方案

**问题本质**: 大量到期卡片 → 用户焦虑 → 回避行为 → 更多积压 → 彻底放弃

**行业最佳实践**（来源：MemoForge, Anki Manual）:

1. **暂停新卡**: 积压 > 每日限额 2 倍时，自动暂停新卡引入（设新卡数为 0）
2. **时间预算制**: 不按卡片数量，按时间预算（如"今天学 20 分钟"），到时间即停
3. **过滤甲板**: 允许按"逾期天数"筛选，优先处理最紧急的卡片
4. **卡片质量审计**: 多数积压源于低质量卡片——自动标记连续失败 ≥3 次的卡片为"水蛭"，建议用户暂停或重写
5. **心理安全设计**: 
   - 不显示"你有 347 张到期卡片"这种恐吓数字
   - 改为"今天的复习任务：约 15 分钟"
   - 使用进度条而非绝对数字

### 1.4 每日限制的心理学依据

| 参数 | 推荐值 | 依据 |
|------|--------|------|
| 每日新卡 | 5-25 张（按时间预算） | 每张新卡会产生 6-8 次未来复习；30 张/天 → 2 周后 200+ 复习/天 |
| 复习上限 | 100-250 张 | 超过 60 分钟注意力显著下降（Pomodoro 原理） |
| 父级甲板限制 | 推荐启用 | 防止单科目占用全部时间预算 |

**来源**: MemoForge Blog, Anki Manual Deck Options

### 1.5 Learning Steps 的科学基础

**Ebbinghaus 遗忘曲线数据**（1885 年首次测量，2015 年 PLOS ONE 复现验证）:

| 时间点 | 遗忘比例 |
|--------|----------|
| 20 分钟 | 42% |
| 1 小时 | 56% |
| 9 小时 | 64% |
| 1 天 | 67% |
| 6 天 | 75% |

**公式**: R = e^(-t/S)，其中 R=记忆保持率，t=时间，S=记忆稳定性

**Learning Steps 设计原理**: 
- **1min**: 验证"刚刚看到的内容是否进入短时记忆"
- **10min**: 短时记忆 → 中期记忆转化的关键窗口
- **1day**: 睡眠巩固后的首次检索——最陡峭遗忘发生在前 24 小时
- 每次成功检索提升 S 值，使记忆更抗衰减

**来源**: Ebbinghaus (1885), Murre & Dros (2015, PLOS ONE), Nature Reviews Psychology (2022)

### 1.6 与当前项目的差距

| 维度 | 当前状态 | 差距 |
|------|---------|------|
| SRS 算法 | 有 Anki 制卡能力（TaskDashboardPage） | 无内置 SRS 调度引擎；依赖 Anki 外部应用 |
| 到期提醒 | 无 | 完全缺失 |
| 每日限制 | 无 | 无上限控制 |
| Learning Steps | 无 | 无步骤化学习流程 |
| 复习债务处理 | 无 | 无积压管理机制 |

### 1.7 改进建议

**P0 — 内置 SRS 引擎**:
- 集成 FSRS 算法（开源 Rust 实现可用）
- 在 Learning Hub 增加"今日复习"入口
- 系统托盘显示到期卡片数

**P1 — 时间预算模式**:
- 用户设置每日学习时间而非卡片数
- 倒计时进度条替代卡片计数器
- 到时间自动结束，显示学习摘要

**P2 — 积压恢复向导**:
- 检测积压 > 2× 每日限额时，弹出恢复向导
- 选项：暂停新卡 / 延长时间 / 筛选优先复习
- 标记水蛭卡并建议重写

## 2. 学习闭环设计原则

### 2.1 完整学习闭环的理论基础

**学习→测试→反馈→复习→巩固**的闭环基于三大理论：

1. **检索练习效应**（Retrieval Practice Effect）: 从记忆中主动提取信息比被动重读有效 150%（Nature Reviews Psychology, 2022）
2. **RMF 模型**（Retrieval-Monitoring-Feedback）: 提取 → 自我监控 → 反馈修正 → 重学正确答案——产生持久长期记忆（Springer, Rawson & Dunlosky）
3. **间隔交错效应**: 将不同主题的练习穿插进行，比集中练习单一主题效果更好

**LOOM 框架**（2025, arXiv）: 通过 LLM 日常对话构建动态学习记忆图，实现个性化长期掌握——验证了"持续性闭环"比"一次性学习"的优势。

### 2.2 为什么"制卡后不自动进入复习"是致命的闭环断裂

**行为科学分析**:

```
理想闭环: 学习内容 → 制卡 → 立即首次复习 → 标记为"学习中" → 进入SRS队列
当前断裂: 学习内容 → 制卡 → ??? → 用户忘记卡片存在 → 从未复习
```

**原因**:
1. **意图-行动鸿沟**（Intention-Action Gap）: 制卡时用户有复习意图，但离开制卡界面后意图消散
2. **缺少即时强化**: 首次成功检索是建立"这张卡片有用"信念的关键——跳过等于放弃锚定
3. **行业对比**: RemNote 制卡后自动弹出 "Practice Now?" 提示；Anki 新卡自动进入当日队列

**竞品行为**:

| 应用 | 制卡后行为 |
|------|-----------|
| Anki | 新卡自动加入当天学习队列 |
| RemNote | 制卡即复习，内嵌在笔记流中 |
| Quizlet | 创建卡组后直接跳转"学习"模式 |
| SuperMemo | 自动根据优先级排入队列 |
| **Deep Student** | **制卡 → 任务完成 → 无后续** |

### 2.3 错题→错题本→复习→掌握的自动化流程

**推荐设计**:

```
[用户答错]
    ↓ 自动
[标记为"错题"，记录错误类型]
    ↓ 自动
[加入错题本，按知识点分组]
    ↓ 定时触发（1天/3天/7天）
[推送错题复习任务]
    ↓ 用户复习
[答对] → 移至"已掌握"
[再答错] → 重置间隔，分析薄弱点
```

**关键设计点**:
- 错题自动标记，无需用户手动收藏
- 错题复习间隔应比正常卡片更短（1d → 3d → 7d vs 1d → 3d → 7d → 21d）
- 每次错题复习提供"变式题"（题干略有变化），防止机械记忆

### 2.4 被动学习→主动回忆的转化路径

| 被动行为 | 转化触发 | 主动行为 |
|---------|---------|---------|
| 阅读 PDF | AI 识别关键概念 | 自动生成填空卡 |
| 翻译文本 | 遇到生词标记 | 一键制作词汇卡 |
| 浏览知识导图 | 点击节点 | 弹出"测试自己"按钮 |
| 听课/笔记 | 笔记完成后 | 提示"基于笔记出题" |
| 高亮标注 | 积累 ≥5 个高亮 | 提示"将高亮转为复习卡" |

### 2.5 与当前项目的差距

| 维度 | 当前状态 | 差距 |
|------|---------|------|
| 制卡→复习 | 制卡后无跳转 | **致命断裂** |
| 错题管理 | 题库有错题标记 | 无自动复习调度 |
| PDF→卡片 | 无 | 无高亮→制卡路径 |
| 翻译→制卡 | 无 | 无生词一键制卡 |
| 知识导图→测试 | 无 | 无节点级测试 |

### 2.6 改进建议

**P0 — 制卡后自动进入首次复习**:
- 制卡完成后弹出"立即复习 (推荐)" / "稍后复习"双按钮
- 默认选中"立即复习"
- 首次复习采用 Learning Steps（1min 后再次出现）

**P0 — 错题自动化闭环**:
- 练习结束时，错题自动加入"错题复习队列"
- 错题复习采用缩短间隔的 SRS
- 连续 3 次答对 → 标记为"已掌握"，移出错题本

**P1 — 被动→主动转化提示**:
- PDF 阅读超过 10 分钟后，底部横幅提示"基于今天的阅读生成复习卡"
- 翻译界面增加"收藏到词汇卡"按钮
- 知识导图节点右键菜单添加"测试这个概念"

## 3. 首次使用引导（Onboarding）

### 3.1 复杂应用的 Onboarding 策略对比

| 策略 | 适用场景 | 效果 | 代表应用 |
|------|---------|------|---------|
| **Tooltip Tour** | 功能少（<5个核心功能） | 完成率低（30-40%）；用户常跳过 | 传统 SaaS |
| **互动式引导** | 核心流程明确 | 完成率高（60-70%）；学以致用 | Duolingo, Canva |
| **空状态引导** | 工具类应用 | 自然学习；但覆盖面有限 | Notion, Obsidian |
| **个性化问卷 + 预设** | 用户群差异大 | 首次体验针对性强 | Notion（"你是谁？你需要什么？"） |
| **连续 Onboarding** | 功能极其丰富 | 持续学习；避免一次性信息过载 | Obsidian Ideaverse |

**来源**: Candu.ai (Notion 分析), Rob Haisfield (Continuous Onboarding), UX Planet

### 3.2 Notion 的 6 条 Onboarding 法则

1. **个性化入口**: 问"你是学生、教师、还是自学者？"→ 定制初始模板
2. **价值先行**: 展示应用能解决的问题，而非功能列表
3. **渐进式暴露**: 首次只展示 3 个核心功能（80/20 法则：80% 用户只用 20% 功能）
4. **提供起点**: 预填充示例内容，让用户"修改"而非"从零创建"
5. **上下文帮助**: 在用户首次触达某功能时提供 Tooltip，而非一次性教完
6. **持续引导**: 使用进度追踪（"你已完成 3/5 个入门步骤"）

### 3.3 渐进式功能暴露（Progressive Disclosure）

**认知负荷理论**: 一次呈现的信息项 ≤ 7±2（Miller's Law）；渐进式暴露减少 30-40% 认知负荷（UX Patterns for Nostr Apps, 2025）。

**三级暴露模型**:
```
Level 1 (首次使用): 核心 3 功能 → AI 聊天, 学习资源, 做题
Level 2 (使用 3 天后): 解锁 → 翻译, 作文批改, 知识导图
Level 3 (使用 1 周后): 全功能 → 制卡任务, 模板管理, 技能管理
```

### 3.4 空状态设计原则

**核心公式**（Material Design, Carbon Design System, Firefox Acorn）:
```
标题（积极语气，避免"无"字） + 1-2 句说明 + 品牌插图 + 行动按钮（强动词 + 名词）
```

**示例**:
- 错误: "暂无卡片"
- 正确: "开始你的记忆之旅 — 从 PDF 中划线即可创建第一张复习卡" [开始制卡]

### 3.5 与当前项目的差距

| 维度 | 当前状态 | 差距 |
|------|---------|------|
| 首次引导 | 无任何 Onboarding 流程 | 完全缺失 |
| 空状态设计 | 部分视图有空状态文字 | 无插图、无引导按钮 |
| 渐进式暴露 | 侧边栏 6 项全部可见 | 无层级暴露 |
| 个性化 | 无 | 无用户角色识别 |

### 3.6 改进建议

**P0 — 首次使用向导**:
- 首次启动弹出 3 步向导：
  1. "你是谁？" → 学生 / 考研党 / 自学者 / 教师
  2. "你主要想做什么？" → 复习备考 / 阅读学习 / 翻译写作 / 全部
  3. 根据选择预填模板和推荐功能顺序

**P1 — 空状态 CTA 设计**:
- 每个空视图添加：品牌插图 + 引导文案 + 主操作按钮
- 学习资源空状态: "拖入你的第一份 PDF，开始智能学习" [导入文件]
- 题库空状态: "从 AI 聊天中让助手出题，或导入已有题库" [让 AI 出题]

**P2 — 持续 Onboarding**:
- 用户首次进入某功能时，显示该功能的 3 句话简介
- "入门进度"卡片：展示"你已探索 3/7 个功能"
- 每周提示一个"你可能不知道的功能"

## 4. 学习激励与游戏化

### 4.1 Streak（连续学习）的心理学效应

**核心机制**: 损失厌恶（Loss Aversion）——人们对失去已拥有的东西的恐惧是获得等价物的快乐的 2 倍。

**Duolingo 实验数据**（来源：Duolingo Blog, NBER 2025 研究）:
- 7+ 天连续学习的用户，次日继续学习的概率是普通用户的 **2.4 倍**
- 将 Streak 与每日目标**解耦**（只需完成 1 课即可延续），14 天留存率提升 **3.3%**
- 里程碑动画（如 7 天、30 天、100 天庆祝）使新用户 7 天留存提升 **1.7%**
- 提供 Streak Freeze（冻结保护）平衡了承诺感与灵活性

**NBER 2025 研究**: 对 60,000 名学生的实验表明，强调连续性的消息显著提升了在线数学平台使用量和数学成绩。

### 4.2 外在动机挤出效应

**传统观点**: 外在奖励会削弱内在动机（Deci & Ryan, 自我决定理论）

**2025 最新研究修正**（Science Direct）:
- 外在奖励可作为"入口"（entry point），帮助用户克服初始惰性
- 关键在于**动机转化**（Motivation Transformation）：从"为了徽章学"逐渐转向"为了学而学"
- 转化不是自动的——需要设计支持

**游戏化误用风险**（arXiv, 语言学习应用研究）:
- 用户过度关注游戏元素而非学习本身
- 竞争性和过度玩乐是两大成因
- 解决方案：将游戏元素与学习成果**绑定**（如"掌握 10 个概念"而非"获得 10 个经验值"）

### 4.3 成就系统的正确设计

**原则**: 成就应反映**真实学习里程碑**，而非活动量。

| 成就类型 | 好的设计 | 坏的设计 |
|---------|---------|---------|
| 知识掌握 | "已掌握 50 个知识点" | "已做 500 道题" |
| 坚持习惯 | "连续 7 天学习" | "本周登录 7 次" |
| 深度理解 | "作文评分从 C 到 A" | "提交了 10 篇作文" |
| 成长轨迹 | "正确率从 60% 提升到 85%" | "获得 1000 经验值" |

### 4.4 学习进度可视化最佳实践

| 可视化类型 | 优势 | 适用场景 | 代表应用 |
|-----------|------|---------|---------|
| **热力图（日历式）** | 直观展示活跃度，GitHub 风格 | 习惯追踪 | GitHub, Anki |
| **进度条/环形图** | 目标感强，易于理解 | 课程完成度 | Duolingo, Coursera |
| **知识树/技能图** | 展示知识关系和掌握层次 | 学科知识体系 | Khan Academy, Skill Tree |
| **雷达图** | 多维能力对比 | 综合能力评估 | 各类测评系统 |
| **趋势折线** | 展示进步轨迹 | 成绩变化 | 学习类 App |

**来源**: AICoursify (热力图 7 步法), Research Square (注意力知识追踪), Mastery Grids (社交进度可视化)

### 4.5 与当前项目的差距

| 维度 | 当前状态 | 差距 |
|------|---------|------|
| 打卡日历 | ✅ LearningHeatmapChart | 已有 GitHub 风格热力图 |
| 连续天数 | 部分实现 | 无 Streak Freeze，无里程碑动画 |
| 成就系统 | 无 | 完全缺失 |
| 进度可视化 | 热力图 + 部分雷达图 | 缺少知识掌握进度、趋势图 |
| 激励反馈 | 无 | 无即时正反馈（答对动画等） |

### 4.6 改进建议

**P0 — Streak 完善**:
- 增加 Streak Freeze 机制（每周 1 次免费冻结）
- 里程碑庆祝：7天/30天/100天/365天 → 展示动画 + 分享卡片
- 将 Streak 与"每日目标"解耦：完成任意 1 个学习活动即延续

**P1 — 知识掌握进度**:
- 按学科/章节展示掌握百分比（进度环）
- "已掌握 / 学习中 / 待学习" 三态卡片视图
- 弱项高亮：正确率 <60% 的知识点标红

**P2 — 轻量成就系统**:
- 5-8 个核心成就（非膨胀式徽章）
- 每个成就绑定真实学习里程碑
- 不设排行榜（避免外在竞争挤出内在动机）

### 5.3 信息架构重组建议

**当前问题**: 6 个扁平导航项缺乏层级关系；"技能管理"和"模板库"对普通用户意义不明。

**推荐分组**:
```
📚 学习（主要）
  ├── AI 聊天            ← 已有 chat-v2
  ├── 学习资源            ← 已有 learning-hub（内含 PDF、笔记、翻译、导图）
  └── 练习与复习          ← 需新增：整合题库、错题本、SRS 复习

🛠 工具（次要）
  ├── 制卡任务            ← 已有 task-dashboard
  └── 模板库              ← 已有 template-management

⚙ 系统（底部固定）
  ├── 设置                ← 已有 settings
  └── 技能管理            ← 已有 skills-management
```

### 5.4 Cmd+K 命令面板的定位

**来源**: Chameleon, Mobbin, HashBuilds, MacStories

**结论**: Cmd+K **不能替代**良好的导航结构，但能**弥补**导航不足。

**数据**:
- 70-80% 用户**从不使用**键盘快捷键
- Cmd+K 是**补充**手段，不是**替代**手段
- 对高级用户极其有价值：跨功能跳转、模糊搜索、快速操作

**当前项目**: 已有 CommandPalette 实现。建议增强为**统一入口**——支持搜索视图、搜索文件、搜索功能、执行操作。

### 5.5 与当前项目的差距

| 维度 | 当前状态 | 差距 |
|------|---------|------|
| 侧边栏层级 | 扁平 6 项 | 无分组，缺乏层级 |
| 隐藏功能 | PDF 等通过其他路径进入 | 不可发现 |
| Cmd+K | ✅ 已有命令面板 | 需要增强内容搜索 |
| 功能分组 | 无 | "学习"和"工具"未区分 |

### 5.6 改进建议

**P0 — 导航分组**:
- 侧边栏增加分组标签（"学习"、"工具"、"系统"）
- 高频功能上移：AI 聊天第一，学习资源第二，练习第三
- "技能管理"和"模板库"移到底部或折叠

**P1 — 增加"练习与复习"导航项**:
- 作为一级入口，整合：题库练习 + 错题复习 + SRS 到期复习
- 显示数字徽标（如"3 张到期"）

**P2 — 命令面板增强**:

## 6. 多模式练习系统的 UX 设计

### 6.1 当前项目的 8 种练习模式

来自 `PracticeModeSelector.tsx` 和 `questionBankStore.ts`:

1. 顺序练习 (sequential)
2. 随机练习 (random)
3. 错题优先 (mistake-first)
4. 标签筛选 (tag-based)
5. 限时练习 (timed)
6. 模拟考试 (mock-exam)
7. 每日一练 (daily)
8. 智能组卷 (smart)

### 6.2 选择悖论分析

**Hick's Law**: 选项数量每翻倍，决策时间增加约 50%。8 个平铺模式需要用户理解每个模式的区别。

**行业对比**:
| 应用 | 练习模式数 | 展示方式 |
|------|-----------|---------|
| Anki | 1（自动调度） | 无需选择 |
| Quizlet | 4（学习/测试/配对/写出） | 标签页切换 |
| Duolingo | 1（自适应） | 完全隐藏选择 |
| Khan Academy | 2（练习/测验） | 简单按钮 |

### 6.3 简化策略

**推荐方案：2 级模式选择**

```
默认入口（1 键启动）:
  🧠 "开始学习" ← 智能推荐模式，自动选择最佳练习内容
  
展开更多（折叠区域）:
  ├── 📋 顺序练习
  ├── 🔀 随机练习
  ├── ❌ 错题复习
  ├── 🏷️ 按标签
  ├── ⏱️ 限时模式
  └── 📝 模拟考试
```

**"每日一练"不应是模式**——应该是首页的快捷入口卡片。
**"智能组卷"应该是默认行为**——重命名为"开始学习"，作为唯一推荐入口。

### 6.4 智能推荐能否替代手动选择

**可以替代 80% 的场景**。

**算法逻辑**:
1. 优先级 1: 到期 SRS 复习卡
2. 优先级 2: 最近错题（按遗忘曲线排序）
3. 优先级 3: 新题（按知识图谱中的薄弱点）
4. 优先级 4: 巩固题（已掌握但即将到期）

**保留手动选择的场景**: 考前模拟（需要真实计时）、按章节专项训练（备考特定范围）。

### 6.5 练习结束后结果展示最佳实践

**必须包含的 4 个元素**:

1. **即时总结**: 正确率 + 用时 + 与上次对比
2. **错题回顾**: 逐题展示错误原因和正确答案
3. **进步可视化**: "你比上次提高了 15%"或"这个知识点从 60% → 80%"
4. **下一步行动**: "复习错题" / "继续练习" / "今日任务已完成"

**不应包含**: 排名/对比（产生焦虑）、过度赞美（降低可信度）

### 6.6 与当前项目的差距

| 维度 | 当前状态 | 差距 |
|------|---------|------|
| 模式数量 | 8 种平铺 | 选择过多，无默认推荐 |
| 智能推荐 | 有"智能组卷"模式 | 不是默认入口 |
| 结果展示 | 未知 | 需要检查 |
| 每日入口 | 有"每日一练"模式 | 应提升为首页卡片 |

### 6.7 改进建议

**P0 — 默认"开始学习"按钮**:
- 将"智能组卷"提升为默认的大按钮入口
- 其余模式折叠在"更多选项"中
- 减少决策疲劳

**P1 — 智能推荐算法增强**:
- 综合考虑：SRS 到期 + 错题频率 + 知识图谱弱点
- 目标：用户只需点"开始学习"，系统自动安排最优练习

**P1 — 结果页面设计**:
- 练习完成后显示：正确率 + 用时 + 知识点掌握度变化
- 突出"进步"而非"错误"
- 提供明确的下一步按钮

## 7. 跨功能集成的 UX 模式

### 7.1 行业最佳实践

**RemNote 模式**: PDF 阅读 → 侧边栏划线 → 自动生成双向链接闪卡 → 内嵌 SRS
**SmarterHumans 模式**: 浏览器/PDF/视频 → 高亮 → 深度链接回源材料 → 复习时一键跳回
**Kosmik 模式**: 多源 PDF → 画布组织 → 提取内容保留源追踪

**共同原则**: **最小化"发现内容"到"创建复习材料"之间的摩擦**

### 7.2 翻译→制卡的交互设计

```
[翻译界面]
  用户选中生词 "ephemeral"
    ↓
  翻译结果弹出: "ephemeral — 短暂的"
  右下角小按钮: [+ 加入词汇卡]
    ↓ 点击
  Toast 提示: "已添加到词汇卡 ✓ — 今日新增 3 词"
  （后台自动创建 SRS 卡片，无需跳转）
```

**关键**: 不跳转、不弹窗、不打断翻译流程。操作反馈用 Toast 即可。

### 7.3 PDF 高亮→笔记→卡片的转化路径

```
[PDF 阅读器]
  用户高亮一段文字
    ↓
  高亮工具栏出现 3 个图标:
    📝 添加笔记  |  🃏 制作卡片  |  📋 复制
    ↓ 点击"制作卡片"
  内联编辑区: 自动填充高亮内容为"答案"
  用户输入"问题"（或 AI 自动生成）
    ↓ 确认
  卡片创建成功，显示来源页码标记
  卡片与 PDF 页面双向链接
```

**参考**: RemNote Reader 分屏设计、SmarterHumans 深度链接

### 7.4 知识导图→复习卡片的交互设计

```
[知识导图]
  用户查看某个概念节点（如"细胞分裂"）
    ↓
  节点悬浮菜单:
    🧠 测试自己  |  📝 添加笔记  |  🔗 查看关联
    ↓ 点击"测试自己"
  弹出迷你卡片:
    Q: "细胞分裂的两种主要类型是什么？"
    [显示答案] ← 用户点击
    A: "有丝分裂和减数分裂"
    [记住了 ✓] [没记住 ✗]
```

### 7.5 不增加复杂度的暴露策略

| 策略 | 说明 | 示例 |
|------|------|------|
| **上下文操作** | 仅在相关上下文中显示跨功能按钮 | PDF 中才显示"制卡"按钮 |
| **渐进暴露** | 首次使用时不显示高级集成 | 用户制作第 5 张卡后才提示"你知道可以从 PDF 直接制卡吗？" |
| **右键菜单** | 将跨功能操作放在右键菜单中 | 选中文本 → 右键 → "翻译 / 制卡 / 添加到导图" |
| **AI 建议** | AI 聊天中主动建议跨功能操作 | "我注意到你在学这个主题，要我基于今天的笔记生成复习卡吗？" |

### 7.6 与当前项目的差距

| 维度 | 当前状态 | 差距 |
|------|---------|------|
| 翻译→制卡 | 无 | 缺少一键制卡 |
| PDF→卡片 | 无高亮→制卡路径 | 无集成 |
| 导图→测试 | 无 | 无节点级交互 |
| AI 建议 | AI 聊天有工具调用能力 | 未利用 AI 做主动建议 |
| 统一右键菜单 | 无 | 无跨功能右键菜单 |

### 7.7 改进建议

**P0 — 选中文本上下文菜单**:
- 任何文本选中后，显示浮动工具栏: [翻译] [制卡] [高亮]
- 这是**最高 ROI 的集成点**——覆盖 PDF、笔记、翻译三个场景

**P1 — PDF 高亮→制卡**:
- PDF 高亮后在工具栏增加"制卡"按钮
- AI 自动生成问题（用户可编辑）
- 卡片保留 PDF 页码来源链接

**P2 — AI 主动建议**:
- 检测用户在某主题上的活动模式
- 在合适时机建议跨功能操作
- 例："你已经阅读了 3 篇关于机器学习的文档，要生成一套复习卡吗？"
